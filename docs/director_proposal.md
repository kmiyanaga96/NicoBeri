# 児童託児所 デジタル化推進プロジェクト：システム設計・導入の概要

本資料は、当施設の業務効率化を目的としたデータベース及びシステムの新規構築にあたり、施設長様にご確認いただきたい「導入の目的」「解決される課題」「セキュリティ・プライバシー保護の仕組み」をまとめたものです。

---

## 1. なぜデジタル化を行うのか？（現状の課題）
現在、当事業所では日々の預かり時間やスケジュールの管理を、**紙のスケジュール表**と**Excel/スプレッドシートへの手入力**で行っております。この運用には以下の課題がありました。
1. **業務負担とミス**: 紙からパソコンへ毎月手入力で転記するため、行政へ給付金を請求する際の「サービス提供実績時間」の計算に多大な時間と労力がかかっています。
2. **状況把握の遅れ**: 当日に誰が来るのか、キャンセルの連絡が入っているのかを全スタッフが即座に共有する手段が乏しい状態です。
3. **限界が近い物理スペース**: 過去の書類やカルテの保管場所が物理的に限界に近づいています。

## 2. システム導入によるメリットと解決策
これらの課題を、新たに構築する事業所専用の「デジタルシステム（Webアプリ）」によって以下の通り解決します。

* **ボタン一つで児童の登降園時間を記録（打刻機能）**
  * スタッフの皆様個人のスマートフォンから、「〇〇くん、到着」「〇〇ちゃん、退出」というボタンを押すだけで記録が完了します。
* **月末の集計作業を完全自動化**
  * 日々の打刻データから、給付金請求に必要な「サービス提供実績記録表」をシステムが自動で集計し、**現在と同じExcelのフォーマットで一括出力**します。手作業による転記・集計は不要になります。
* **スマホで「今日の予定」を一目で確認**
  * その日の来所予定とキャンセル状況を、スマートフォンやパソコンの画面で一覧表示できる「ダッシュボード（掲示板）」を設置します。

## 3. 児童のプライバシーとセキュリティ（最重要事項）
個人のスマートフォンからアクセスできるようになる反面、「情報漏洩」や「プライバシー侵害」を完全に防ぐための堅牢なセキュリティをシステムの根幹に設けています。

### 段階的なアクセス制御（誰が何を見れるか）
スタッフの権限を「管理者（施設長・システム担当者）」と「一般スタッフ」に分けます。
1. **一般スタッフ（スマホ利用）**
   * スマホからは**「今日の予定確認」と「到着・退出の打刻」のみ**が許可されます。
   * 児童の住所や電話番号、アレルギー情報、医療的ケアが必要なカルテや、過去の書類などは**一切開けません（システム側で完全にブロックされます）**。
2. **管理者・施設長（事業所PC利用）**
   * 所長様とシステム担当者（アカウント２名のみ）は、個人情報であるデジタルカルテや書類アーカイブの閲覧が可能です。

### インフラ・データの保管場所について
* クラウド上の非常に安全なデータセンター（Supabaseという最新のデータベース基盤）にデータが暗号化されて保管されます。
* 物理的なサーバーを事業所に置かないため、機械の故障によるデータ消失の心配がありません。万が一災害が起きても、データは安全に守られます。

## 4. 今後の進め方
まずは本番運用に向けて、**「児童のスケジュール管理・打刻・日報自動出力」** という、もっとも負担の大きい作業をピンポイントで解決するシステム（初期バージョン）から開発をスタートします。

不要に複雑な機能を詰め込まず、現場の皆様が無理なく使えるシンプルさを最優先事項として開発を進めさせていただきます。
