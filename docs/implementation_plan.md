# 児童託児所（障害児日中一時預かり所）デジタル化計画

現状のアナログ業務（紙のスケジュール表、スプレッドシートへの手入力等）をデジタル化し、業務効率化とスタッフの負担軽減を実現しつつ、児童のプライバシーを保護するシステムを構築します。

## 解決すべき課題とアプローチ

1. **サービス提供実績時間の算定と実績記録表作成の自動化**
   - **現状**: 手書きのスケジュール表からスプレッドシートへ毎月手入力しており、実績時間の算定に手間がかかっている。
   - **解決策**: システム上で児童の「到着」「退出」時間を打刻（記録）し、月末に指定された書式（ExcelやCSV、スプレッドシート等）で自動出力・集計する。行政の算定ロジックをシステムに組み込む。
2. **当日の利用予定児童の把握・キャンセルの管理**
   - **現状**: スタッフが当日の予定を把握しづらく、キャンセルがLINEのみで管理されている。
   - **解決策**: 日付ごとに「来所予定の児童リスト」を一覧表示するダッシュボード機能。LINE Botや専用フォームからキャンセル連絡を受けると自動で予定表に反映される仕組みを作る。
3. **書類保管容量の限界**
   - **現状**: 紙の書類が物理スペースを最大まで圧迫している。
   - **解決策**: アナログ書類（基本情報や日報、アレルギー情報など）をデータベース化。過去の書類もスキャンしてクラウドストレージにセキュアに保管。
4. **従業員のスマホからのアクセスとプライバシー保護の両立**
   - **現状**: 事務所のWindows PCからのみアクセス可能だが、従業員のスマホからも確認したい。児童のプライバシー保護が最優先。
   - **解決策**:
     - **堅牢な認証**: 多要素認証（MFA）の導入や、安全な認証プロバイダを用いたログイン機構。
     - **権限管理（RBAC）**: 「管理者」「一般スタッフ」で閲覧・編集できるデータを制限する。
     - **段階的アクセス制御（スマホとPCの使い分け）**: 児童の詳細な個人情報やカルテは**事業所のネットワーク（特定のIPアドレス）やマスターPCからのみ閲覧可能**とし、従業員のスマホからは「当日の予定一覧確認」と「出退勤・打刻」のみ可能にするといった段階的なアクセス制御を設ける。

## System Architecture & Tech Stack

> [!NOTE]
> **確定した技術スタックとインフラ**
> - **フロントエンド**: Next.js (React) + TailwindCSS
> - **バックエンド / DB**: Supabase
> - **ホスティング**: Vercel
>
> クラウドの利用制約は特にないため、開発スピードとスマホアクセスの利便性に有利な上記クラウド構成で進めます。プライバシー保護については、データベースの「行レベルセキュリティ(RLS)」等を用いて、スマホからの不要なデータアクセスを堅牢に遮断します。

## MVP（初期リリース版）のスコープ定義

初期段階での複雑化を防ぎ、システムを現場に定着させるため、まずは「児童の管理」に特化した機能に絞って開発します。

**【今回の最優先範囲（MVP）】**
1. **スケジュール把握**: 当日および翌日に預かる児童名簿の全体把握機能。
2. **打刻機能**: 児童の預かり開始・終了時間のスマホからの記録。
3. **権限管理**: 児童のカルテ等の機密情報はスマホから閲覧不可とし、PCからのみアクセス可能とするセキュリティ。
4. **記録自動出力**: 月末におけるサービス提供実績記録表用のデータ自動集計・出力機能。

**【将来の拡張（今回はスコープ外）】**
- スタッフ（従業員）側の人事機能全般（スタッフ自身の出退勤打刻、給与体系との連携など）。
- ※これらは児童側のシステム運用が定着した後、第2フェーズ以降での追加を検討します。

## Proposed Features (機能要件)

### スタッフ用（スマホからのアクセスを想定）
- **ログイン・認証**: ID/パスワード（および多要素認証）。
- **当日スケジュール一覧**: 本日来所予定の児童一覧、キャンセル・遅刻状況のリアルタイム確認。
- **打刻機能**: 児童の到着時間、退出時間の入力。

### 事務管理者用（事業所のWindows PCからのアクセスを想定）
- **児童情報管理（デジタルカルテ）**: 児童の基本情報、アレルギー、配慮事項などの管理（スマホからは原則閲覧不可またはマスキング）。
- **スタッフ・アカウント管理**: スタッフの追加・権限付与。アクセスログの確認。
- **実績記録表データの出力**: 月次の給付金請求に必要な打刻時間集計データを、指定のフォーマットでエクスポートする。
- **過去書類アーカイブ**: スキャンしたPDF等のアップロード・管理機能。

### 保護者用連携（オプション・将来拡張）
- LINE公式アカウントを用いた「欠席・遅刻連絡ボット」。保護者がLINEから連絡すると、システム側のスケジュールに自動で「キャンセル」等が反映される。（※要件定義完了後、初期フェーズに含めるかを検討）

## Verification Plan

### 自動検証（システムテスト）
- **打刻・集計テスト**: 仮想データを用いた出退勤入力を行い、月末の集計データ（滞在時間、サービス提供時間）が行政の算定ロジックと完全に一致するかをユニットテストで検証する。
- **アクセス制御テスト**:
  - 一般スタッフ権限のアクセスでは、高度な機密情報（カルテなど）を取得するAPIがエラーを返すことを検証。
  - 特定のIP（事業所のみ）以外からアクセスした場合のブロック機能をテスト（IP制限を導入する場合）。

### 手動検証（ユーザー受入テスト）
- **UI検証**: 実際のWindows PCとスマホブラウザの両方を用いて、ダッシュボードの視認性、打刻のしやすさを確認していただく。
- **ファイル出力**: 実際に出力されたCSVやExcelデータが、既存の「サービス実績記録表」の仕様を満たしているかを確認していただく。
